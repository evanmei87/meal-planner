# shadcn/ui Foundation (Tailwind 4 + Base UI + design tokens) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Upgrade the `web/` frontend to Tailwind 4, adopt shadcn/ui on the Base UI engine with documented design tokens, and re-skin the four shared components onto the token system without changing their public APIs.

**Architecture:** Tailwind 4 (CSS-first config via `@theme`, `@tailwindcss/vite` plugin) is the styling base. shadcn/ui copies component source into `web/src/components/ui/`; semantic color/spacing/typography tokens live as CSS variables in `web/src/index.css` and are mapped to utilities via `@theme inline`. The four existing shared components keep their props and JSX but swap hard-coded colors for token classes. An `@/` import alias points at `web/src`.

**Tech Stack:** React 18, TypeScript 5, Vite 6, **Tailwind CSS 4**, shadcn/ui (Base UI engine), `class-variance-authority`, `clsx`, `tailwind-merge`, Vitest 4 + React Testing Library + MSW.

## Global Constraints

- **Tailwind 4, not 3.** The app is being upgraded from v3; do not leave v3 config or directives behind.
- **Full shadcn default aesthetic.** `baseColor: neutral`, near-neutral primary. The green brand accent retires from the four components.
- **`@/` alias everywhere.** Every relative cross-module import under `web/src` is converted to `@/…`.
- **The four components' public props are unchanged:** `Card({ children, className? })`, `Table({ columns, rows })` with `Column = { key: string; header: string; render?: (value: unknown, row: Record<string, unknown>) => React.ReactNode }`, `ErrorBanner({ message })`, `Spinner()` (no props).
- **Button is generate-only.** Added via the CLI to prove the workflow; not wired into any feature page.
- **No backend changes.** Work is confined to `web/` and docs.
- **Gates:** `npm test -- --run` and `npx tsc --noEmit` pass from `web/` at the end of every task.
- **Base UI engine** is the shadcn primitive engine (not Radix).

## Context for a fresh engineer

- **Repo layout:** Python backend in `src/`, React SPA in `web/`. You only touch `web/` and `docs/`.
- **Working branch:** `feat/41-shadcn-ui-foundation` (already checked out). Run all `npm`/`npx` commands **from the `web/` directory**; run `git` from the repo root. The shell is Windows PowerShell — run one command per line (no `&&` chaining).
- **Run tests:** from `web/`, `npm test -- --run` (Vitest, jsdom). **Type-check:** `npx tsc --noEmit`. **Build:** `npm run build`. **Dev server:** `npm run dev` (Vite at :5173; the backend is not needed for the four-tab smoke check because pages render loading/empty states without it).
- **Tests assert on text and ARIA roles, not class names** — so the upgrade and re-skin are safe as long as rendered text/roles are preserved.
- **Spec:** `docs/superpowers/specs/2026-06-26-shadcn-ui-foundation-design.md`.

# Design Tokens

> Foundation for issue #41. Tokens are CSS variables in `web/src/index.css`, mapped to Tailwind utilities via `@theme inline`. Consume these tokens — do not re-derive raw colors in feature code.

## Color — semantic (shadcn, neutral base)

Standard shadcn tokens drive the app surface. Use the Tailwind utilities they generate:

| Token utility | Use for |
|---|---|
| `bg-background` / `text-foreground` | page surface + body text |
| `bg-card` / `text-card-foreground` | card surfaces |
| `bg-muted` / `text-muted-foreground` | subdued fills + secondary text |
| `bg-primary` / `text-primary-foreground` | primary actions and **active/selected state** |
| `bg-accent` / `text-accent-foreground` | hover/active accents |
| `border-border`, `bg-input`, `ring-ring` | borders, inputs, focus rings |
| `text-destructive`, `bg-destructive` | errors / destructive actions |

**Active/selected state:** map to `bg-primary text-primary-foreground`. #28 (today highlight), #38 (selected day), and the nav active link all converge on this token when their issues migrate. (Those screens are out of scope for #41 and still use hard-coded green for now.)

## Color — exercise-type palette (for #38)

Each type has a strong value (border) and a subtle tint (background):

| Type | Strong (`--color-exercise-<type>`) | Subtle (`-subtle`) |
|---|---|---|
| running | green-600 | green-50 |
| walking | blue-500 | blue-50 |
| biking | orange-500 | orange-50 |
| swimming | cyan-500 | cyan-50 |
| strength | purple-500 | purple-50 |

## Spacing & shape

- **Spacing:** use Tailwind's default spacing scale (`p-2`, `gap-3`, `space-y-6`, …). No custom spacing variables.
- **Radius:** `--radius` (shadcn default) with `rounded-sm` / `rounded-md` / `rounded-lg` derived from it.

## Typography

- **Font:** `--font-sans` (system sans stack from Tailwind 4 / shadcn default).
- **Type scale:** Tailwind's default `text-xs … text-2xl` with their paired line-heights. Headings in the app use `text-lg font-semibold`; body uses `text-sm`.

export interface Column {
  key: string
  header: string
  render?: (value: unknown, row: Record<string, unknown>) => React.ReactNode
}

interface TableProps {
  columns: Column[]
  rows: Record<string, unknown>[]
}

export function Table({ columns, rows }: TableProps) {
  return (
    <div className="overflow-x-auto">
      <table className="min-w-full text-sm border-collapse">
        <thead>
          <tr className="bg-muted">
            {columns.map((col, index) => (
              <th
                key={`${col.key}-${index}`}
                className="px-3 py-2 text-left font-semibold text-muted-foreground"
              >
                {col.header}
              </th>
            ))}
          </tr>
        </thead>
        <tbody>
          {rows.map((row, i) => (
            <tr key={i} className="border-t border-border hover:bg-muted/50">
              {columns.map((col, colIndex) => (
                <td key={`${col.key}-${colIndex}`} className="px-3 py-2 text-foreground">
                  {col.render
                    ? col.render(row[col.key], row)
                    : String(row[col.key] ?? '')}
                </td>
              ))}
            </tr>
          ))}
          {rows.length === 0 && (
            <tr>
              <td
                colSpan={columns.length}
                className="px-3 py-4 text-center text-muted-foreground text-sm"
              >
                No data
              </td>
            </tr>
          )}
        </tbody>
      </table>

  )
}
